# “我”页：用户名/头像/密码修改接口说明

本文档描述前端进入“我”页面后，进行用户资料展示与修改的后端接口。

> 说明：当前项目未做登录态/JWT 鉴权，中台统一通过前端传 `user_id` 来识别用户（与收藏/偏好等接口保持一致）。
> 
> 响应约定：HTTP 状态码统一返回 200，业务通过 `code` 表示结果。

---

## 1. 获取用户信息（用于“我”页展示）

- 方法：`GET`
- 路径：`/api/user/me`
- Query 参数：
  - `user_id`：必填，用户 ID（数字）

### 请求示例

`GET /api/user/me?user_id=1`

### 响应示例

成功：
```json
{
  "code": 200,
  "msg": "获取成功",
  "data": {
    "user_id": 1,
    "username": "test01",
    "avatar_url": "http://host/static/avatars/u1/20260316/xxx.jpg"
  }
}
```

失败（user_id 格式错误）：
```json
{ "code": 400, "msg": "user_id 格式错误" }
```

失败（用户不存在）：
```json
{ "code": 404, "msg": "用户不存在" }
```

---

## 2. 修改用户名（登录账号，唯一）

- 方法：`PUT`
- 路径：`/api/user/username`
- Content-Type：`application/json`

### Body

```json
{
  "user_id": 1,
  "username": "新的用户名"
}
```

### 规则

- 用户名不能为空。
- 长度不超过 50 个字符。
- 不能包含空格/制表符/换行等空白字符。
- 用户名全局唯一；若已被占用返回 `code=409`。

### 注意事项

- 用户名修改成功后，用户的 `user_id` 不会改变。
- 后续登录时需要使用新的 `username`；前端如有缓存用户名，请同步更新本地缓存。

### 响应示例

成功：
```json
{ "code": 200, "msg": "修改成功", "data": { "username": "新的用户名" } }
```

失败（用户名不合法/空）：
```json
{ "code": 400, "msg": "用户名不合法" }
```

失败（用户名已存在）：
```json
{ "code": 409, "msg": "该用户名已存在，换一个试试吧~" }
```

---

## 3. 修改登录密码

- 方法：`PUT`
- 路径：`/api/user/password`
- Content-Type：`application/json`

### Body

```json
{
  "user_id": 1,
  "old_password": "旧密码",
  "new_password": "新密码"
}
```

### 规则

- 必须传旧密码用于校验。
- 新密码至少 6 位。

### 响应示例

成功：
```json
{ "code": 200, "msg": "修改成功" }
```

失败（旧密码错误）：
```json
{ "code": 401, "msg": "旧密码错误" }
```

失败（新密码不合法）：
```json
{ "code": 400, "msg": "新密码不合法（至少 6 位）" }
```

---

## 4. 修改头像（上传图片并更新用户头像 URL）

- 方法：`POST`
- 路径：`/api/user/avatar`
- Content-Type：`multipart/form-data`

### 表单字段

- `user_id`：必填
- 文件字段（三选一即可）：
  - `avatar`（推荐）
  - `image`
  - `file`

### 文件限制

- 仅支持：JPG / JPEG / PNG

### 行为说明

- 上传新头像后会更新 `avatar_url` 指向新头像。
- 为避免服务器存储堆积，后端会删除旧头像对应的本地文件（仅限本服务生成并存放在 `static/avatars/...` 下的旧头像）。

### 响应示例

成功：
```json
{
  "code": 200,
  "msg": "上传成功",
  "data": {
    "avatar_url": "http://host/static/avatars/u1/20260316/xxx.jpg"
  }
}
```

失败（缺少 user_id）：
```json
{ "code": 400, "msg": "缺少 user_id" }
```

失败（格式不支持）：
```json
{ "code": 400, "msg": "仅支持 JPG、JPEG、PNG 格式的图片" }
```

## 5. 与登录接口的配合（建议）

登录成功后接口 `/api/login` 的返回 `data` 已包含：

- `user_id`
- `username`（登录账号）
- `avatar_url`（头像）

前端可在首次登录后直接展示；进入“我”页时也可调用 `GET /api/user/me` 做一次刷新。

---

## 6. 数据库字段说明

用户表 `users` 新增/使用字段：

- `username`：登录账号（唯一）
- `avatar_url`：头像 URL（默认空字符串）

项目启动时使用 GORM `AutoMigrate`，会自动把新列补到表结构中。

## 7. 前端对接示例（JS/TS）

下面给出一组最小可用的前端调用示例（Web/H5/小程序的 JS 逻辑都可参考）。

### 7.1 基础约定

- 后端总是返回 HTTP 200；请用 `res.code` 判断业务成功（`code === 200`）。
- `user_id` 建议在登录成功后从 `/api/login` 返回的 `data.user_id` 保存到本地（如 localStorage / Pinia / Redux）。

示例约定：

```ts
const API_BASE = "http://你的后端域名或IP:8080";

function assertOK(res: any) {
  if (!res || res.code !== 200) {
    throw new Error(res?.msg || "请求失败");
  }
  return res;
}
```

### 7.2 获取“我”页用户信息

#### fetch 示例

```ts
async function getMe(userId: number) {
  const url = `${API_BASE}/api/user/me?user_id=${encodeURIComponent(String(userId))}`;
  const res = await fetch(url, { method: "GET" });
  const data = await res.json();
  assertOK(data);
  return data.data as {
    user_id: number;
    username: string;
    avatar_url: string;
  };
}
```

### 7.3 修改用户名（登录账号，唯一）

```ts
async function updateUsername(userId: number, username: string) {
  const res = await fetch(`${API_BASE}/api/user/username`, {
    method: "PUT",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ user_id: userId, username })
  });
  const data = await res.json();
  assertOK(data);
  return data.data as { username: string };
}
```

### 7.4 修改密码

```ts
async function updatePassword(userId: number, oldPassword: string, newPassword: string) {
  const res = await fetch(`${API_BASE}/api/user/password`, {
    method: "PUT",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      user_id: userId,
      old_password: oldPassword,
      new_password: newPassword
    })
  });
  const data = await res.json();
  assertOK(data);
  return true;
}
```

### 7.5 上传并更新头像

#### 浏览器/H5（input file）示例

```ts
async function uploadAvatar(userId: number, file: File) {
  const form = new FormData();
  form.append("user_id", String(userId));
  form.append("avatar", file); // 也兼容字段名 image/file

  const res = await fetch(`${API_BASE}/api/user/avatar`, {
    method: "POST",
    body: form
  });
  const data = await res.json();
  assertOK(data);
  return data.data as { avatar_url: string };
}
```

#### axios 示例（如果你项目里已在用 axios）

```ts
import axios from "axios";

const http = axios.create({ baseURL: API_BASE });

async function uploadAvatarAxios(userId: number, file: File) {
  const form = new FormData();
  form.append("user_id", String(userId));
  form.append("avatar", file);

  const { data } = await http.post("/api/user/avatar", form);
  assertOK(data);
  return data.data as { avatar_url: string };
}
```

### 7.6 前端页面最小流程建议

1) 登录成功保存 `user_id`。

2) 进入“我”页：调用 `getMe(user_id)` 渲染 `username`、`avatar_url`。

3) 用户修改：

- 用户名：调用 `updateUsername`，成功后更新 UI。
- 密码：调用 `updatePassword`，成功后提示“修改成功”。
- 头像：先上传 `uploadAvatar`，成功后用返回的 `avatar_url` 替换头像显示。

> 备注：后端返回的 `avatar_url` 是一个完整 URL（由请求的 Host/协议拼出来）。若你的服务在 Nginx/网关后面，请确保转发时 Host 头正确，否则建议前端也可使用其对应的相对路径 `/static/...` 访问静态资源。
