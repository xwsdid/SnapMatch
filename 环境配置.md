# 环境配置指南

本文档详细说明如何将服务器上的 `photo_backend` 项目复制到本地环境并成功运行。

## ⚠️ 重要提示：版本一致性

**服务器实际环境配置**（请务必保持与本地一致）：

| 组件 | 服务器版本 | 本地建议版本 | 兼容性要求 |
|------|-----------|------------|-----------|
| **Go** | 1.25.0 | ≥ 1.25.0 | ⚠️ 必须 ≥ 1.25.0 |
| **MySQL** | 8.0.44 | 8.0.x | ✅ 8.0 系列即可 |
| **操作系统** | Alibaba Cloud Linux 3 | 任意主流系统 | ✅ 无特殊要求 |
| **ONNX Runtime** | 1.16.0 (通过 go module 自动管理) | 1.16.0 | ✅ 版本自动同步 |
| **onnxruntime_go** | 1.27.0 | 1.27.0 | ⚠️ 必须一致 |

**版本不一致的后果**：
- ❌ Go 版本过低 → 编译失败
- ❌ MySQL 版本差异大 → 数据库连接或语法问题
- ❌ onnxruntime_go 版本不同 → ONNX 模型推理失败

## 🔍 版本检查与同步【必读】

### 1. 检查服务器环境版本

在服务器上运行以下命令，记录版本号：

```bash
# 创建环境信息文件
cat > /tmp/env_check.sh << 'EOF'
#!/bin/bash
echo "=== 服务器环境配置 ==="
echo "Go 版本:"
go version
echo ""
echo "MySQL 版本:"
mysql --version
echo ""
echo "操作系统:"
cat /etc/os-release | grep "PRETTY_NAME"
echo ""
echo "项目依赖版本:"
cd /root/photo_backend
cat go.mod | grep -E "^require|^go"
echo ""
echo "ONNX Runtime 模型文件:"
ls -lh models/comp_model.onnx
EOF

chmod +x /tmp/env_check.sh
bash /tmp/env_check.sh > server_env.txt
cat server_env.txt
```

### 2. 保存服务器环境信息

将生成的 `server_env.txt` 下载到本地保存：

```bash
scp root@your-server-ip:/tmp/server_env.txt ~/Downloads/
```

### 3. 本地环境版本同步

#### Go 版本同步
```bash
# 检查本地 Go 版本
go version

# 如果版本低于 1.25.0，必须升级
# 下载地址：https://go.dev/dl/
```

#### 依赖版本同步（关键！）
```bash
# 方法 1: 直接复制 go.mod 和 go.sum（推荐）
# 确保本地的 go.mod 和 go.sum 与服务器完全一致

# 方法 2: 使用 go mod tidy 自动同步
cd /path/to/photo_backend
go mod tidy

# 验证版本一致性
go list -m all | grep onnxruntime_go
# 应该显示：github.com/yalue/onnxruntime_go v1.27.0
```

### 4. 可能出现的版本问题

#### 场景 1：本地 Go 版本过低
```bash
# 错误信息
package photo_backend requires Go 1.25.0 or later

# 解决方案：升级 Go 到 1.25.0+
```

#### 场景 2：依赖版本不一致
```bash
# 检查 go.mod 中的版本
cat go.mod

# 强制使用指定版本
go get github.com/yalue/onnxruntime_go@v1.27.0
go get github.com/gin-gonic/gin@v1.12.0
go mod tidy
```

#### 场景 3：ONNX Runtime 动态库问题
```bash
# onnxruntime_go v1.27.0 会自动下载对应版本的动态库
# 不需要手动安装，但需要确保网络能访问 GitHub

# 如果下载失败，配置代理
export GOPROXY=https://goproxy.cn,direct
go mod tidy
```

## Go 环境安装

### Windows 系统

1. **下载安装包**
   - 访问：https://go.dev/dl/
   - 下载 `go1.25.0.windows-amd64.msi`

2. **安装**
   - 双击安装包，使用默认设置
   - 安装路径：`C:\Program Files\Go`

3. **配置环境变量**
   ```
   # 系统环境变量中添加：
   GOPATH=C:\Users\你的用户名\go
   PATH=%PATH%;C:\Program Files\Go\bin
   PATH=%PATH%;%GOPATH%\bin
   ```

4. **验证安装**
   ```powershell
   go version
   # 应显示：go version go1.25.0 windows/amd64
   ```

5. **安装 C 编译器（CGO 必需，重要！）**

本项目依赖 `onnxruntime_go`，在 Windows 上编译时需要 CGO，因此必须有 `gcc`（否则会出现 `cgo: C compiler "gcc" not found`）。

推荐方式（MSYS2）：
```powershell
# 1) 安装 MSYS2：https://www.msys2.org/
# 2) 打开 “MSYS2 MinGW x64” 终端执行：
pacman -S --needed mingw-w64-x86_64-gcc
```

然后把 `C:\msys64\mingw64\bin` 加到系统 `PATH`，重新打开终端后验证：
```powershell
gcc --version
```

> 如果你们只在 Linux 服务器上编译/运行后端，则服务器上安装 `gcc`/`g++` 即可。

## VLM（拍摄建议/姿势引导）转发配置

Go 后端会把图片转发到 Python VLM 服务（FastAPI）。需要配置 Python 服务地址：

- `VLM_API_URL`：完整推理 URL（优先级最高），例如 `http://127.0.0.1:8000/vlm/infer`
- `VLM_API_BASE_URL`：基础地址（未设置 `VLM_API_URL` 时使用），例如 `http://127.0.0.1:8000`（会自动拼成 `/vlm/infer`）
- `VLM_HTTP_TIMEOUT_SECONDS`：Go → Python 请求超时（默认 120 秒）

生产部署建议：
- Python VLM 服务部署在带 GPU 的机器（内网可访问即可，不建议公网暴露）；
- Go 后端部署在业务服务器，通过内网地址调用 Python；
- 设置 `VLM_API_BASE_URL=http://<gpu-server-ip>:8000`。

### macOS 系统

1. **使用 Homebrew 安装**
   ```bash
   # 安装 Homebrew（如果未安装）
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   
   # 安装 Go
   brew install go@1.25
   
   # 配置环境变量（添加到 ~/.zshrc 或 ~/.bash_profile）
   echo 'export PATH=$(go env GOPATH)/bin:$PATH' >> ~/.zshrc
   source ~/.zshrc
   ```

2. **验证安装**
   ```bash
   go version
   ```

### Linux 系统

1. **Ubuntu/Debian**
   ```bash
   # 方法 1: 使用包管理器（可能版本较旧）
   sudo apt update
   sudo apt install golang-go
   
   # 方法 2: 手动安装最新版本
   wget https://go.dev/dl/go1.25.0.linux-amd64.tar.gz
   sudo tar -C /usr/local -xzf go1.25.0.linux-amd64.tar.gz
   
   # 添加到环境变量
   echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
   echo 'export GOPATH=$HOME/go' >> ~/.bashrc
   echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.bashrc
   source ~/.bashrc
   ```

2. **CentOS/RHEL**
   ```bash
   # 安装 Go
   sudo yum install -y golang
   
   # 或手动安装
   wget https://go.dev/dl/go1.25.0.linux-amd64.tar.gz
   sudo tar -C /usr/local -xzf go1.25.0.linux-amd64.tar.gz
   
   # 添加环境变量
   echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
   source ~/.bashrc
   ```

3. **验证安装**
   ```bash
   go version
   ```

   ```powershell

   sc query MySQL80
   ```

1. **使用 Homebrew 安装**
   ```bash
   # 安装 MySQL
   brew install mysql@8.0
   
   # 启动 MySQL 服务
   brew services start mysql@8.0
   ```

2. **验证**
   ```bash
   mysql --version
   ```

1. **Ubuntu/Debian**
   ```bash
   # 安装 MySQL
   sudo apt update
   sudo apt install mysql-server mysql-client -y
   
   # 启动服务
   sudo systemctl start mysql

   # 安全配置
   ```


## ONNX Runtime 安装

### Windows 系统

1. **下载预编译库**
   - 访问：https://github.com/microsoft/onnxruntime/releases
   - 下载：`onnxruntime-win-x64-1.16.0.zip`

2. **解压并配置**
   ```powershell
   # 解压到合适位置
   Expand-Archive onnxruntime-win-x64-1.16.0.zip -DestinationPath "C:\Program Files\ONNXRuntime"
   
   # 添加环境变量
   # 将以下路径添加到系统 PATH：
   C:\Program Files\ONNXRuntime\lib
   ```

3. **复制 DLL 文件**
   ```powershell
   # 或者直接将 onnxruntime.dll 复制到项目目录
   Copy-Item "C:\Program Files\ONNXRuntime\lib\onnxruntime.dll" "C:\你的项目路径\photo_backend\"
   ```

### macOS 系统

1. **使用 Homebrew 安装**
   ```bash
   brew install onnxruntime
   ```

2. **手动安装**
   ```bash
   # 下载
   wget https://github.com/microsoft/onnxruntime/releases/download/v1.16.0/onnxruntime-osx-arm64-1.16.0.tgz
   
   # 解压
   tar -xzf onnxruntime-osx-arm64-1.16.0.tgz
   
   # 复制动态库
   sudo cp onnxruntime-osx-arm64-1.16.0/lib/libonnxruntime.1.16.0.dylib /usr/local/lib/
   ```

### Linux 系统

1. **Ubuntu/Debian**
   ```bash
   # 下载
   wget https://github.com/microsoft/onnxruntime/releases/download/v1.16.0/onnxruntime-linux-x64-1.16.0.tgz
   
   # 解压
   tar -xzf onnxruntime-linux-x64-1.16.0.tgz
   
   # 进入目录
   cd onnxruntime-linux-x64-1.16.0
   
   # 复制动态库到系统目录
   sudo cp lib/libonnxruntime.so.1.16.0 /usr/lib/
   sudo ln -s /usr/lib/libonnxruntime.so.1.16.0 /usr/lib/libonnxruntime.so
   
   # 更新动态库缓存
   sudo ldconfig
   ```

2. **CentOS/RHEL**
   ```bash
   # 下载并解压（同上）
   wget https://github.com/microsoft/onnxruntime/releases/download/v1.16.0/onnxruntime-linux-x64-1.16.0.tgz
   tar -xzf onnxruntime-linux-x64-1.16.0.tgz
   
   # 复制动态库
   sudo cp onnxruntime-linux-x64-1.16.0/lib/libonnxruntime.so.1.16.0 /usr/lib64/
   sudo ln -s /usr/lib64/libonnxruntime.so.1.16.0 /usr/lib64/libonnxruntime.so
   
   # 更新缓存
   sudo ldconfig
   ```

3. **验证安装**
   ```bash
   # 检查动态库是否存在
   ldconfig -p | grep onnxruntime
   ```

## 数据库配置

### 1. 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 输入密码后，执行：
CREATE DATABASE photography_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户（可选，建议使用专门用户）
CREATE USER 'photo_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON photography_db.* TO 'photo_user'@'localhost';
FLUSH PRIVILEGES;

# 验证
SHOW DATABASES;
USE photography_db;
EXIT;
```

### 2. 修改数据库连接配置

编辑文件：`dao/db.go`

找到第 15 行：
```go
dsn := "root:123456@tcp(127.0.0.1:3306)/photography_db?charset=utf8mb4&parseTime=True&loc=Local"
```

修改为你的实际配置：
```go
# 格式：用户名：密码@tcp(主机：端口)/数据库名？参数
dsn := "root:你的密码@tcp(127.0.0.1:3306)/photography_db?charset=utf8mb4&parseTime=True&loc=Local"

# 或使用专门用户
dsn := "photo_user:your_password@tcp(127.0.0.1:3306)/photography_db?charset=utf8mb4&parseTime=True&loc=Local"
```

### 3. 测试数据库连接

```bash
# 使用命令行测试
mysql -u root -p -h 127.0.0.1 -P 3306 photography_db

# 如果能成功登录，说明配置正确
```

## 常见问题排查

### 1. Go 模块下载失败

**错误**：`dial tcp: lookup github.com: no such host`

**解决方案**：
```bash
# 配置国内镜像
go env -w GOPROXY=https://goproxy.cn,direct

# 或
go env -w GOPROXY=https://goproxy.io,direct

# 清除缓存重新下载
go clean -modcache
go mod tidy
```

### 2. MySQL 连接被拒绝

**错误**：`dial tcp 127.0.0.1:3306: connect: connection refused`

**解决方案**：
```bash
# Windows: 检查服务是否运行
sc query MySQL80
# 如果未运行：
net start MySQL80

# Linux/macOS:
sudo systemctl status mysql
sudo systemctl start mysql
```

### 3. 数据库权限不足

**错误**：`Access denied for user 'root'@'localhost'`

**解决方案**：
```sql
-- 重置 root 密码
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
```

### 4. ONNX Runtime 库找不到

**Windows 错误**：
```
error while loading shared libraries: libonnxruntime.dll: cannot open shared object file
```

**解决方案**：
```powershell
# 复制 DLL 到项目目录
Copy-Item "C:\Program Files\ONNXRuntime\lib\onnxruntime.dll" ".\"

# 或添加到 PATH
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\ONNXRuntime\lib", "User")
```

## 🚨 常见环境差异问题

### 问题 1：本地能编译，服务器编译失败

**症状**：
```bash
# 本地正常
go build -o photo_backend .  # ✅ 成功

# 服务器报错
go build -o photo_backend .  # ❌ 失败
```

**可能原因**：
1. 服务器 Go 版本过低
2. 服务器 `go.mod` 或 `go.sum` 文件不同步
3. 服务器缺少某些系统库

**解决方案**：
```bash
# 1. 检查并升级服务器 Go 版本
ssh root@server "go version"

# 2. 同步 go.mod 和 go.sum
scp go.mod go.sum root@server:/root/photo_backend/

# 3. 在服务器上重新下载依赖
ssh root@server "cd /root/photo_backend && go mod tidy"

# 4. 重新编译
ssh root@server "cd /root/photo_backend && go build -o photo_backend ."
```

### 问题 2：本地能运行，服务器 ONNX 推理失败

**症状**：
```bash
# 本地正常
./photo_backend  # ✅ 启动成功，构图分析正常

# 服务器报错
./photo_backend  # ⚠️ 构图模型初始化失败
```

**可能原因**：
1. 服务器缺少 ONNX Runtime 动态库
2. 模型文件路径错误
3. 模型文件损坏

**解决方案**：
```bash
# 1. 检查动态库
ssh root@server "ldconfig -p | grep onnxruntime"

# 2. 检查模型文件
ssh root@server "ls -lh /root/photo_backend/models/comp_model.onnx"

# 3. 重新下载模型文件（如果损坏）
scp models/comp_model.onnx root@server:/root/photo_backend/models/

# 4. 检查模型路径配置
ssh root@server "grep -n 'InitCompositionService' /root/photo_backend/main.go"
```

### 问题 3：本地和服务器数据库连接不一致

**症状**：
```bash
# 本地正常
✅ 数据库连接与自动迁移成功！

# 服务器报错
❌ 数据库连接失败
```

**解决方案**：
```bash
# 1. 检查服务器数据库配置
ssh root@server "grep 'dsn :=' /root/photo_backend/dao/db.go"

# 2. 检查服务器 MySQL 服务
ssh root@server "systemctl status mysqld"

# 3. 测试服务器数据库连接
ssh root@server "mysql -u root -p -e 'SELECT 1'"

# 4. 如果密码不同，分别配置
# 本地：dao/db.go 使用本地密码
# 服务器：dao/db.go 使用服务器密码
```

### 问题 4：架构差异导致的问题（AMD64 vs ARM64）

**检查架构**：
```bash
# 本地
go env GOARCH

# 服务器
ssh root@server "go env GOARCH"
```

**如果架构不同**：
- ONNX Runtime 会自动下载对应架构的动态库
- 确保 `go.mod` 中 `onnxruntime_go` 版本一致即可
- 不需要手动处理

## ✅ 最佳实践建议

### 1. 使用 Docker 保证环境一致性（强烈推荐）

创建 `Dockerfile`：
```dockerfile
FROM golang:1.25.0-alpine

# 安装运行时依赖
RUN apk add --no-cache mysql-client

# 设置工作目录
WORKDIR /app

# 复制依赖文件
COPY go.mod go.sum ./
RUN go mod download

# 复制源代码
COPY . .

# 编译
RUN go build -o photo_backend .

# 暴露端口
EXPOSE 8080

# 启动服务
CMD ["./photo_backend"]
```

使用 Docker 部署：
```bash
# 本地构建
docker build -t photo_backend:latest .

# 本地运行
docker run -p 8080:8080 photo_backend:latest

# 推送到服务器
docker save photo_backend:latest | ssh root@server "docker load"

# 服务器运行
ssh root@server "docker run -d -p 8080:8080 photo_backend:latest"
```

### 2. 使用部署脚本自动化

创建 `deploy.sh`：
```bash
#!/bin/bash
set -e

echo "=== 开始部署 ==="

# 1. 版本检查
echo "检查 Go 版本..."
GO_VERSION=$(go version | awk '{print $3}')
if [[ "$GO_VERSION" != "go1.25.0" ]]; then
    echo "⚠️  警告：Go 版本不是 1.25.0，当前版本：$GO_VERSION"
fi

# 2. 编译
echo "编译项目..."
go build -o photo_backend .

# 3. 测试
echo "运行测试..."
./photo_backend &
PID=$!
sleep 3
kill $PID 2>/dev/null || true

# 4. 部署到服务器
echo "部署到服务器..."
scp photo_backend root@server:/root/photo_backend/
scp -r models root@server:/root/photo_backend/

# 5. 重启服务
echo "重启服务..."
ssh root@server "systemctl restart photo_backend"

echo "✅ 部署完成！"
```

配置 GitHub Actions 或 GitLab CI，自动完成：
- 版本检查

- 部署到服务器
